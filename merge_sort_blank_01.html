① merge() 함수란?

        
정렬된 두 배열 left와 right를 하나로 합치는 함수입니다.
두 배열의 앞에서부터 하나씩 비교해서 더 작은 값을 result에 추가합니다.

      

      

        
② result, i, j 초기화가 왜 필요한가요?

        
result는 병합 결과를 담을 빈 리스트,
i는 left 배열의 포인터, j는 right 배열의 포인터입니다.
각각 인덱스 0에서 시작합니다.
result, i, j = [], 0, 0

      

      

        
③ while 조건은 무엇인가요?

        
양쪽 배열에 아직 비교할 원소가 남아있는 동안 반복합니다.
i < len(left) and j < len(right)
하나라도 소진되면 while을 빠져나옵니다.

      

      

        
④ if 조건 — 언제 left를 추가하나요?

        
left[i] <= right[j] 일 때
→ left의 값이 더 작거나 같으면 left[i]를 result에 추가하고 i를 1 증가시킵니다.
<= 이어야 안정 정렬(Stable Sort)이 됩니다!

      

      

        
⑤ else — 언제 right를 추가하나요?

        
left[i] > right[j] 일 때 (else 조건)
→ right의 값이 더 작으면 right[j]를 result에 추가하고 j를 1 증가시킵니다.

      

      

        
⑥ while 이후 남은 원소는?

        
한쪽 배열이 먼저 소진되면 while이 끝납니다.
남은 쪽은 이미 정렬된 상태이므로 그대로 이어 붙입니다.
result += left[i:] + right[j:]
둘 중 소진된 쪽의 슬라이싱은 빈 리스트 []가 됩니다.

      

      

        
⑦ mid = len(arr) // 2 는 무엇인가요?

        
배열을 절반으로 나누는 위치입니다.
//는 정수 나눗셈(소수점 버림)입니다.
예: len=6 → mid=3, len=3 → mid=1
arr[:mid] = 왼쪽 절반, arr[mid:] = 오른쪽 절반

      

      

        
⑧ 재귀(Recursion)란 무엇인가요?

        
함수가 자기 자신을 다시 호출하는 것입니다.
merge_sort(arr[:mid]) → merge_sort 안에서 merge_sort를 부릅니다.
원소가 1개가 될 때까지 계속 쪼개고, 1개가 되면 바로 반환합니다 (종료 조건).

      

      

        
⑨ 왜 len(arr) <= 1이면 바로 반환하나요?

        
원소가 0개 또는 1개인 배열은 이미 정렬된 상태입니다.
이것이 재귀의 종료 조건(Base Case)입니다.
종료 조건이 없으면 재귀가 무한히 반복됩니다!
